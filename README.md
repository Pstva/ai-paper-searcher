## ИИ-поисковик научных статей

### Идея проекта

Поиск подходящих научных статей под запрос пользователя.

### Примерное описание работы сервиса

1. Пользователь может загрузить в чат (UI/tg бот) запрос. Запрос может быть как текстом, так и аудио.
2. Затем его запрос обрабатывается. 

+ Если пользователь передал запрос текстом, то он напрямую передается LLM.
LLM проинсруктрирована в системном промте как необходимо сократить запрос/доработать его. Так же у нее есть доступ к MCP tool (предварительно выбрала [этот](https://hub.docker.com/mcp/server/paper-search/overview])).
+ Если пользователь передал аудио в качестве запроса, то сначала это аудио переводится моделью STT (пока что приглянулась [bond005/whisper-podlodka-turbo](https://huggingface.co/bond005/whisper-podlodka-turbo)) в текст, затем тектовый запрос уже передается в LLM (как описано выше)

3. Далее ответ от модели передается пользователю и пользователя просят оценить ответ. 

+ В случае положительной оценки, пользователю предлагают начать новый поиск. Тогда начинается новый чат с чистой историей поиска.
+ В случае отрицательной оценки, пользователю предлагают уточнить запрос или начать новый поиск. В первом случае, начинается новая итерация, но предыдущая история запросов добавляется к запросу (и будет передана модели). Второй случай аналогичен преыдущему пункту.

В [папку](app/models) добавила предварительное описание объектов системы, оно будет дополняться.
